# This test attack and response provided the following key takeaway points:
- when using a cloud-based virtual device (e.g. Azure), a cloud workload protection solution should be enabled (e.g. Defender for Cloud)
- network security rules (e.g. what types of inbound/outbound traffic do we allow and deny) should be understood and applied correctly
- the attack showed how the Microsoft security ecosystem works together. The attack set off the alarm in Defender for Cloud which integrates natively with Defender for Endpoint when enabled. After viewing the alert, I could investigate the details using the investigation package and take the necessary actions.
